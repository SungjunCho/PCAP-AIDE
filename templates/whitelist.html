📖 화이트리스트 추가 방법

global_whitelist.yaml 파일의 whitelist: 섹션 아래에 항목을 추가하고 Reload 버튼을 누르면 서버 재시작 없이 즉시 반영됩니다.


description — 설명 (사람이 읽기 위한 메모)
source — 출처 (OS명, 툴명, 장비명 등)
protocols — 적용 프로토콜 목록. 생략하거나 ~ 이면 모든 프로토콜
match_type — 매칭 방식 (4가지 중 하나 선택)
  payload_exact — payload 전체 hex 완전 일치
  payload_hex   — payload 앞부분 hex starts-with 매칭
  payload_text  — payload 내 문자열 포함 여부 (부분 매칭)
  type_code    — ICMP type/code 조합만으로 정상 판정
value — 매칭할 값 (hex 문자열 또는 일반 문자열)
icmp_type / icmp_code — ICMP 전용 필터 (생략 시 모든 값)

# 1) ICMP — payload text 포함 매칭 - description: "사내 모니터링 시스템 ICMP 헬스체크" source: "Internal NMS" protocols: [ICMP] icmp_type: 8 icmp_code: 0 match_type: payload_text value: "HEALTHCHECK" # 2) ICMP — payload hex prefix 매칭 (장비 고유 패턴) - description: "RouterA 고유 ICMP echo payload" source: "RouterA" protocols: [ICMP] icmp_type: 8 icmp_code: 0 match_type: payload_hex value: "deadbeef01020304" # 3) ICMP — type/code 조합만으로 정상 판정 (제어 메시지) - description: "ICMP TTL Expired — 모든 payload 정상" source: "RFC 792 traceroute" protocols: [ICMP] icmp_type: 11 match_type: type_code value: "" # 4) HTTP — 특정 정상 응답 패턴 - description: "사내 API 서버 정상 응답" source: "Internal API" protocols: [HTTP] match_type: payload_text value: "HTTP/1.1 200 OK" # 5) 모든 프로토콜 공통 (protocols 생략) - description: "전체 프로토콜 공통 정상 패턴" source: "Internal" match_type: payload_text value: "NORMAL-TRAFFIC"
